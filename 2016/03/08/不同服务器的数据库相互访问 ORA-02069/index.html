不同服务器的数据库相互访问 ORA-02069

实现两个不同服务器的数据库 severA,serverB。 serverA要访问serverB对象 ,serverB把数据写入serverA的对象,以及要注意的事项;

1.在serverA 上用 userC 创建一个tableD 用来储存 serverB 过来的数据;
2.userF 通过 DBLINK链接到serverB;

create public database link DBLINK
 connect to userF
 using 'serverA';

3.在userC 对tableD 做相关赋权给 userF; 

grant select,insert, update on tableD to userF;

4.在userF 创建tableD 的同义词；

create or replace synonym tableD for userC.tableD;

5.在serverB的userG上通过DBLINK创建tbaleD 的同义词； 

create or replace synonym tableD for userC.tableD@DBLINK;

实现在userG上写数据后userC可以即时读取，再通过tableD上的触发器来实现所需的操作；

注意：
1.在第4步需要确认是否能访问tableD，相关权限是否正常。如果漏掉这步，userG 将无权限访问tableD,或提示对象不存在；
2.如果在userG用户上，用userG用户下的序列直接插数据到 tbaleD，会报错，提示ORA-02069: “此操作的 global_names 参数必须设置为 TRUE”； 应该先拿到序列值，在把数值插入tableD;
